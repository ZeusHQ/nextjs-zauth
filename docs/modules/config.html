ConfigParameters

ConfigParameters: DeepPartial<BaseConfig & NextConfig>

Configuration properties.

The Server part of the SDK can be configured in 2 ways.

1. Environmental Variables

The simplest way to use the SDK is to use the named exports (HandleAuth, HandleLogin, HandleLogout, HandleCallback, HandleProfile, GetSession, GetAccessToken, WithApiAuthRequired and WithPageAuthRequired), eg:

// pages/api/auth/[...zidentity].js
import { handleAuth } from '@zeushq/nextjs-zidentity';

return handleAuth();

When you use these named exports, an instance of the SDK is created for you which you can configure using environmental variables:

Required
  • ZIDENTITY_SECRET: See secret
  • ZIDENTITY_ISSUER_BASE_URL: See issuerBaseURL
  • ZIDENTITY_BASE_URL: See baseURL
  • ZIDENTITY_CLIENT_ID: See clientID
  • ZIDENTITY_CLIENT_SECRET: See clientSecret

Optional
  • ZIDENTITY_CLOCK_TOLERANCE: See clockTolerance
  • ZIDENTITY_HTTP_TIMEOUT: See httpTimeout
  • ZIDENTITY_ENABLE_TELEMETRY: See enableTelemetry
  • ZIDENTITY_IDP_LOGOUT: See idpLogout
  • ZIDENTITY_ID_TOKEN_SIGNING_ALG: See idTokenSigningAlg
  • ZIDENTITY_LEGACY_SAME_SITE_COOKIE: See legacySameSiteCookie
  • NEXT_PUBLIC_ZIDENTITY_LOGIN: See NextConfig.routes
  • ZIDENTITY_CALLBACK: See BaseConfig.routes
  • ZIDENTITY_POST_LOGOUT_REDIRECT: See BaseConfig.routes
  • ZIDENTITY_AUDIENCE: See BaseConfig.authorizationParams
  • ZIDENTITY_SCOPE: See BaseConfig.authorizationParams
  • ZIDENTITY_ORGANIZATION: See NextConfig.organization
  • ZIDENTITY_SESSION_NAME: See SessionConfig.name
  • ZIDENTITY_SESSION_ROLLING: See SessionConfig.rolling
  • ZIDENTITY_SESSION_ROLLING_DURATION: See SessionConfig.rollingDuration
  • ZIDENTITY_SESSION_ABSOLUTE_DURATION: See SessionConfig.absoluteDuration
  • ZIDENTITY_COOKIE_DOMAIN: See CookieConfig.domain
  • ZIDENTITY_COOKIE_PATH: See CookieConfig.path
  • ZIDENTITY_COOKIE_TRANSIENT: See CookieConfig.transient
  • ZIDENTITY_COOKIE_HTTP_ONLY: See CookieConfig.httpOnly
  • ZIDENTITY_COOKIE_SECURE: See CookieConfig.secure
  • ZIDENTITY_COOKIE_SAME_SITE: See CookieConfig.sameSite

2. Create your own instance using InitZeusIdentity

If you don't want to configure the SDK with environment variables or you want more fine grained control over the instance, you can create an instance yourself and use the handlers and helpers from that.

First, export your configured instance from another module:

// utils/zidentity.js
import { InitZeusIdentity } from '@zeushq/nextjs-zidentity';

export default InitZeusIdentity({ ...ConfigParameters... });

Then import it into your route handler:

// pages/api/auth/[...zidentity].js
import zidentity from '../../../../utils/zidentity';

return zidentity.handleAuth();

Note If you use InitZeusIdentity, you should not use the other named exports as they will use a different instance of the SDK.
